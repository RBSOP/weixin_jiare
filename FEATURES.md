# 微信视频号加热平台数据采集 - 功能文档

## 一、功能概览

| 功能模块 | 说明 |
|----------|------|
| 账号管理 | 多账号添加、cookie 保存、打开账号浏览器 |
| 订单采集 | 订单列表、再来一单配置、订单详情、电商效果、人群分析 |
| 数据展示 | 分页查询、筛选、排序、分类展示 |
| 数据导出 | Excel 导出（含截图） |
| 配置管理 | 消耗阈值、时间范围、浏览器模式、分辨率 |

## 二、账号管理

### 2.1 添加账号

1. 在「账号管理」页点击「添加账号」
2. 弹出新浏览器窗口，打开微信视频号加热平台登录页
3. 使用微信扫码登录
4. 登录成功后自动跳转首页，提取账户名、视频号
5. 保存 cookie 至 `data/account_cookies/<account_id>.json`
6. 同名账号会更新而非新建

**说明**：cookie 保存后，后续采集可免扫码，直到会话过期。

### 2.2 打开账号浏览器

选择已添加账号，点击「打开浏览器」，使用该账号 cookie 打开加热平台首页，便于手动操作或验证。

### 2.3 删除账号

点击账号卡片上的「删除」，将删除账号记录及 cookie 文件，订单的 `account_id` 置空。

## 三、订单采集

### 3.1 采集流程

1. 选择账号
2. 可选：设置开始时间、结束时间（格式 `YYYY-MM-DD` 或 `YYYY-MM-DD HH:MM:SS`）
3. 点击「开始采集」
4. 程序使用该账号 cookie 打开订单列表页
5. 按页采集：每页筛选出「已完成」「消耗>阈值」「时间范围内」的订单
6. 对未采集详情链的订单，依次执行：
   - 再来一单配置
   - 订单详情
   - 电商加热效果
   - 人群分析

### 3.2 采集范围与筛选

| 筛选条件 | 说明 |
|----------|------|
| 订单状态 | 仅采集「已完成」订单 |
| 消耗金额 | 仅采集消耗 > `COST_MIN`（默认 500 元）的订单 |
| 时间范围 | 可选，按订单创建时间过滤 |
| 增量采集 | 默认开启，遇已入库订单停止翻页，避免重复 |

### 3.3 采集内容明细

#### 订单列表（orders）

- 加热主播、创建时间、名称、编号、目标、状态、预算、出价/ROI、实际加热时长、消耗、ROI

#### 再来一单配置（order_create_config）

- **选择加热类型**：加热对象类型、加热订单类型
- **选择加热对象**：主播昵称
- **选择加热方案**：预计带来金额、订单名称、加热方式、放量模式、成交ROI、加热素材
- **预算与时间**：订单预算、加热时长
- **人群定向**：定向类型、观众性别、粉丝层推荐、名单推荐、观众年龄、设备、城市、兴趣
- **其他**：其他支付方式

#### 订单详情（order_detail_data）

- **加热信息**：订单名称、编号、加热目标、状态、预算、预计时长、下单/开始/结束时间、实际加热时长、加热出价、放量模式、人群定向
- **消耗进度**：消耗微信豆、预算微信豆、消耗进度
- **直播间加热效果**：消耗总金额、曝光总人数、进入总人数、点赞总次数、评论总次数、新增总关注
- **十分钟级数据**：时间序列，每时间点含消耗、曝光、观看、点赞、评论、新增粉丝

#### 电商加热效果（order_ecommerce_statistic）

- **自选指标**：总消耗、直接成交金额、直接成交订单数、直接成交ROI、直播间曝光人数、短视频曝光次数、直播间观看人数、直播间商品点击率、千次展示费用、总成交ROI、间接成交金额、直播间观看人次
- **数据明细**：加热主播与创建时间、名称与编号、加热开始/结束时间、实际加热时长、直播间进入率、GPM、直播间转化率、直播间转化成本

#### 人群分析（order_people_statistic）

- **人群漏斗**：直播间曝光人数、观看人数、商品曝光人数、商品点击人数、总下单人数、总成交人数，及各转化率
- **人群分布**：观众商品偏好、八类人群占比、性别分布、年龄分布、地域分布

### 3.4 页面截图

采集过程中会对以下页面截图并保存至 `data/screenshots/<promotion_id>/`：

- `create_config.png` - 再来一单配置页
- `order_detail.png` - 订单详情页
- `ecommerce.png` - 电商加热效果页
- `people.png` - 人群分析页

## 四、数据展示

### 4.1 数据展示页

- **分页**：支持分页，每页条数可调
- **筛选**：按消耗阈值、订单编号、主播昵称、订单名称筛选
- **排序**：按创建时间、消耗、状态等排序
- **分类展示**：数据按「订单基本信息」「再来一单配置」「加热信息」「直播间加热效果」「十分钟级数据」「电商加热效果」「人群漏斗」「人群分布」「截图」等分类展示，不同分类使用不同颜色

### 4.2 十分钟级数据

每个订单可能有多条十分钟级时间点数据，展示时每时间点一行，其他列复用订单信息。

### 4.3 截图查看

点击截图列中的「查看」可加载对应页面截图。

## 五、数据导出

### 5.1 Excel 导出

- 按当前筛选条件导出
- 列顺序与页面展示一致，按分类分组
- 表头使用分类颜色区分
- 十分钟级数据：每时间点一行
- 截图：嵌入 Excel 单元格（小图预览）

### 5.2 导出格式

- 文件格式：`.xlsx`
- 文件名：`export_YYYYMMDD_HHMMSS.xlsx`

## 六、配置管理

### 6.1 消耗阈值（COST_MIN）

在 `config.py` 中修改，默认 500 元。仅采集消耗大于该值的订单。

### 6.2 时间范围

发起采集时可选填开始时间、结束时间，不填表示全量。

### 6.3 UI 配置（data/ui_config.json）

| 配置项 | 说明 | 可选值 |
|--------|------|--------|
| mode | 浏览器模式 | `prod` 无头 / `dev` 有头 |
| resolution | 窗口分辨率 | 如 `1280x800` |

可通过数据展示页设置，或直接编辑 JSON 文件。

### 6.4 全量重采（FULL_RESYNC）

在 `config.py` 中设为 `True` 时，订单列表不按已入库 ID 提前停止，会翻到底。用于全量重新采集。

## 七、技术特性

- **增量采集**：遇已存在订单即停止翻页，减少重复请求
- **断点续传**：中断后重跑，已采集订单跳过，未采集订单继续
- **API 拦截**：通过 Playwright 监听响应，拦截接口数据，无需解析 DOM
- **DOM 兜底**：部分字段 API 不完整时，从页面 DOM 提取
- **弹窗采集**：观众兴趣、观众城市、粉丝层推荐等需点击弹窗才能获取完整数据
- **多账号**：支持多账号独立 cookie，按账号发起采集

## 八、使用场景

1. **定期采集**：按周/月采集订单数据，用于报表分析
2. **效果复盘**：导出 Excel，分析消耗、ROI、人群分布等
3. **配置复用**：通过「再来一单配置」了解历史订单的定向、出价等设置
4. **多账号管理**：企业多子账号统一采集、统一展示
